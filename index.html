Unreal Engine Runtime Plugin

MultiplayerDebugTools

Lightweight, in-viewport multiplayer diagnostics for PIE, standalone, and packaged builds. Designed for fast iteration while validating network conditions.

Quick Start Blueprint API

At a glance

  • Toggle key: F10
  • Console command: mdt.Toggle
  • Overlay: SMultiplayerDebugOverlay
  • Data provider: UMultiplayerDebugSubsystem
  • Supported engines: UE 4.27 → UE 5.7
  • Version: 1.0.0

What It Shows

MetricDescription
RoleStandalone, Client, Listen Server, Dedicated Server
FPSSampled from GAverageFPS
Ping (ms)Average ping across active connections
Loss In / Out (%)Inbound / outbound packet loss percentage
Recv / SentBandwidth with automatic B/s, KB/s, MB/s formatting
Actors / Net ActorsTotal world actors and replicated actor count

If there is no active network connection, network metrics are displayed as N/A.

Installation

  1. Place MultiplayerDebugTools under your project Plugins/ folder.
  2. Open your project in Unreal Editor.
  3. Enable Multiplayer Debug Tools from Edit > Plugins.
  4. Restart the editor.

Quick Usage

  • Toggle overlay with F10.
  • Or run mdt.Toggle in console.

PIE multiplayer validation flow

  1. Set Number of Players to 2 or more.
  2. Choose Listen Server play mode.
  3. Launch PIE and enable overlay in each window.
  4. Compare role/ping/loss/bandwidth values side-by-side.

Blueprint API

Access UMultiplayerDebugSubsystem from Game Instance subsystem utilities. The subsystem exposes runtime controls, snapshots, and update events.

Functions

  • ToggleOverlay()
  • IsOverlayVisible() -> bool
  • GetStatsSnapshot() -> FMultiplayerDebugStatsSnapshot
  • RefreshStatsNow()
  • SetStatsUpdateInterval(float InSeconds)
  • GetStatsUpdateInterval() -> float
  • GetFeatureToggles() -> FMultiplayerDebugFeatureToggles
  • SetFeatureToggles(FMultiplayerDebugFeatureToggles)
  • SetFeatureEnabled(EMultiplayerDebugMetric Feature, bool bEnabled)
  • IsFeatureEnabled(EMultiplayerDebugMetric Feature) -> bool
  • SetAllFeaturesEnabled(bool bEnabled)

Event Dispatcher

OnStatsUpdated(FMultiplayerDebugStatsSnapshot Snapshot)

Feature Enum

Role, FPS, Ping, PacketLoss, Bandwidth, ActorCounts

Snapshot Fields

EnabledFeatures, NetRole, FPS, PingMs, PacketLossInPercent, PacketLossOutPercent, BandwidthInBytesPerSecond, BandwidthOutBytesPerSecond, TotalActors, ReplicatedActors, bHasNetworkConnection

C++ Usage

#include "MultiplayerDebugSubsystem.h"

if (UGameInstance* GI = GetGameInstance())
{
    if (UMultiplayerDebugSubsystem* DebugSub = GI->GetSubsystem<UMultiplayerDebugSubsystem>())
    {
        DebugSub->ToggleOverlay();
        DebugSub->SetFeatureEnabled(EMultiplayerDebugMetric::Bandwidth, true);
        DebugSub->SetStatsUpdateInterval(0.2f);
    }
}

DebugSub->OnStatsUpdated.AddDynamic(this, &UMyObject::HandleStatsUpdated);

Runtime Behavior

  • Default stats interval: 0.25s (minimum 0.05s).
  • Actor counts update internally every 1.0s.
  • Overlay timer refreshes every 0.25s.
  • Client side sampling uses ServerConnection.
  • Server side sampling aggregates ClientConnections.
  • UE_SERVER dedicated targets skip Slate overlay and input handling.

Module Architecture

FMultiplayerDebugToolsModule

Registers the mdt.Toggle command and input preprocessor for F10.

UMultiplayerDebugSubsystem

Collects/cache stats, controls overlay visibility, broadcasts updates.

SMultiplayerDebugOverlay

Renders values with formatting and threshold-aware color hints.

Data flow

  1. Input/console triggers overlay toggle.
  2. Subsystem creates/removes viewport widget.
  3. Ticker refreshes cached snapshot.
  4. Overlay active timer redraws displayed values.

Build & Dependencies

  • Public: Core, CoreUObject, Engine, InputCore
  • Private (non-server): Slate, SlateCore

Add "MultiplayerDebugTools" to your module dependencies if you include plugin headers.

Limitations & Notes

  • Overlay requires an active viewport.
  • F10 may conflict with other input handlers.
  • Non-network sessions will show network metrics as N/A.
  • Packet-loss values come from cumulative packet counters per sample.

Troubleshooting

Overlay not visible

  • Ensure plugin is enabled and editor restarted.
  • Try mdt.Toggle manually in the console.

F10 not responding

  • Confirm command works via mdt.Toggle.
  • Check if any other system consumes F10.

Ping/loss/bandwidth show N/A

  • Run networked sessions (listen server + clients, or connected client/server).

No overlay on dedicated server

  • Expected: dedicated server builds do not render a Slate viewport overlay.
